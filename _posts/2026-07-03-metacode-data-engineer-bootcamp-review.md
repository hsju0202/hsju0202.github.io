## 내 상황과 고민

스타트업에서 2년간 백엔드 개발자로 일하면서 정말 다양한 업무를 경험했다. 회사 규모가 크지 않다 보니 프론트엔드, 백엔드, 인프라, 데이터 업무를 가리지 않고 맡아야 했고, 그 과정에서 자연스럽게 데이터 플랫폼 구축과 데이터 파이프라인 개발 업무의 비중이 커졌다.

문제는 대부분의 기술이 처음이었다는 점이다. 기존에는 Java와 Spring을 주로 사용했는데, 데이터 플랫폼을 구축하면서 Spark, Airflow, Iceberg 같은 도구를 새로 다뤄야 했다. 어떻게든 필요한 기능은 만들었지만, 시간이 지날수록 "이게 정말 잘 구축한 구조일까?", "실무에서는 이런 문제를 어떻게 해결할까?"라는 의문이 계속 남았다.

회사 안에는 데이터 엔지니어링 관점에서 질문할 사람이 많지 않았기 때문에 고민이 더 커졌다. 지금 내가 만든 데이터 플랫폼과 파이프라인을 점검하고, 현업 데이터 엔지니어들은 어떤 기준으로 설계하고 운영하는지 알고 싶어서 메타코드 데이터 엔지니어 부트캠프를 수강하기로 했다.

## 부트캠프를 선택한 이유

가장 큰 이유는 실무자에게 직접 물어볼 수 있는 시간이 있다는 점이었다. 나는 현재 회사에서 구축한 데이터 플랫폼과 파이프라인이 적절한 구조인지 궁금했고, 다른 회사에서는 데이터 플랫폼을 어떻게 설계하고 운영하는지도 알고 싶었다. 메타코드 부트캠프에는 실무자 특강과 Q&A 세션, 현직 데이터 엔지니어와 함께하는 스터디가 있어서 이런 고민을 직접 질문할 수 있을 것 같았다.

제공되는 강의 구성도 마음에 들었다. Spark, Airflow, Kubernetes, AWS 등 현재 회사에서 사용하거나 구축해본 기술 스택과 겹치는 내용이 많았고, 그동안 실무를 하며 부족하다고 느꼈던 기초를 다시 정리할 수 있을 것 같았다.

재직 중에 참여할 수 있는지도 중요한 기준이었다. 강의와 과제는 일주일 안에 자유롭게 진행할 수 있었고, 라이브 특강과 스터디는 평일 밤이나 주말에 진행되어 직장을 다니면서도 참여할 수 있었다. 출석, 강의 수강, 과제, 스터디를 성실히 수행하면 비용을 환급받을 수 있다는 점도 부담을 줄여주는 요소였다.

## 부트캠프 활동 내용

### [메타코드 부트캠프] 강의 수강

![메타코드 부트캠프 강의 수강 화면](/assets/img/metacode-bootcamp/photo1.png)

강의는 파이썬 기초부터 SQL, Git, AWS, k8s, airflow, spark, kafka 등 정말 다양한 기술 스택의 강의가 제공된다.

이러한 기술들을 간단한 데이터 파이프라인을 설계하고 개발할 수 있는 정도까지 학습하게 된다. 다루는 기술이 많다보니 각 기술에 대해 핵심만 학습하고 더 깊은 학습은 부족하다고 느꼈지만 그거는 수강생이 추후 실무에서나 프로젝트에서나 부족한 부분을 스스로 채워야하는 부분이라고 생각한다.

### [메타코드 부트캠프] 진도 및 과제 수행표

![메타코드 부트캠프 과제 수행 화면](/assets/img/metacode-bootcamp/photo3.png)


위 사진처럼 매주 강의 진도율, 과제 제출 여부, 점수가 함께 기재된다. 과제는 그 주에 수강한 강의 내용을 바탕으로 퀴즈나 미션 형태로 제공된다.

퀴즈와 과제는 채점 대상이고, 점수는 환급 여부에도 영향을 준다. 처음에는 부담이 있었지만 강의를 잘 듣고 내용을 꼼꼼히 정리하면 충분히 풀 수 있는 문제들이었다. 오히려 과제를 풀면서 내가 강의를 제대로 이해했는지 확인할 수 있어서 도움이 됐다.

### [메타코드 부트캠프] 현직 데이터 엔지니어 라이브 특강

![메타코드 부트캠프 라이브 특강 화면](/assets/img/metacode-bootcamp/photo2.png)

현직 데이터 엔지니어분들이 다양한 주제로 라이브 특강을 진행해주신다. 특강 참여가 필수는 아니지만, 가능하다면 꼭 듣는 것을 추천하고 싶다. 강의만 들을 때보다 데이터 엔지니어링을 바라보는 시야가 훨씬 넓어졌다.

특강 마지막에는 Q&A 시간이 있어서 평소 궁금했던 점을 질문할 수 있었다. 데이터 파이프라인을 실무에서는 어떻게 설계하는지, 장애나 성능 이슈를 어떤 식으로 바라보는지 같은 질문을 할 수 있어서 좋았다.

내가 들었던 특강은 다음과 같은 주제로 진행됐다.

- 데이터 엔지니어 직무와 커리어
- 규모에 따라 진화하는 데이터 아키텍처
- 파이프라인 로드테스트
- Kafka를 활용한 실시간 데이터와 배치 데이터 개선

## 현직 데이터 엔지니어 라이브 스터디

부트캠프를 진행하면서 가장 좋았던 시간은 현직 데이터 엔지니어와 함께한 라이브 스터디였다. 4주 동안 매주 2회, 특정 주제를 가지고 Zoom으로 스터디를 진행했다.

스터디는 데이터 수집, 정제, 처리, BI까지 이어지는 실시간 데이터 파이프라인 아키텍처를 먼저 그려보고, 매 시간 한 단계씩 함께 구현하는 방식으로 진행됐다. 단순히 코드를 따라 치는 시간이 아니라, 각 단계마다 "왜 이 기술을 선택했는지", "주의할 점은 무엇인지", "장애가 나면 어떻게 복구할 것인지", "데이터가 제대로 수집되고 있는지 어떻게 확인할 것인지"를 함께 이야기했다.

이 과정에서 데이터 엔지니어가 실무에서 고민해야 하는 지점이 생각보다 훨씬 많다는 것을 알게 됐다. 특히 현재 회사에서는 데이터 수집 모니터링, 데이터 신선도, 데이터 품질 관리 같은 부분을 깊게 다뤄본 적이 없었는데, 스터디를 통해 필요성을 크게 느꼈고 실제로 회사에서도 구현해봐야겠다고 생각했다.

### 실습 결과 정리

스터디를 하면서 구현한 내용을 정리하다 보니, 단순히 강의를 듣는 것보다 훨씬 오래 기억에 남았다. 데이터가 어디서 들어오고, 어떤 과정을 거쳐 처리되며, 최종적으로 어떤 형태로 활용되는지 직접 연결해보는 경험이 좋았다.

### 오프라인 발표 준비

스터디 마지막에는 오프라인으로 만나 각자 진행한 내용을 발표하는 시간도 있었다. 발표 자료는 멘토님이 꾸준히 첨삭해주셨고, 덕분에 단순한 과제 제출물이 아니라 추후 포트폴리오로도 활용할 수 있을 정도로 정리할 수 있었다.

## 추천하고 싶은 이유

데이터 엔지니어링 관련 교육은 다른 개발 직군에 비해 많지 않다. 특히 직장이나 학교를 병행하면서 참여할 수 있는 교육은 더 찾기 어렵다. 그런 점에서 메타코드 부트캠프는 재직자도 비교적 현실적으로 참여할 수 있는 과정이었다.

물론 직장을 다니면서 부트캠프를 병행하는 일이 쉽지만은 않았다. 그래도 돌아보면 얻은 것이 훨씬 많았다. 강의와 과제를 통해 기본기를 정리할 수 있었고, 특강과 스터디를 통해 실무에서 데이터 엔지니어가 어떤 관점으로 문제를 바라보는지 배울 수 있었다.

환급 제도도 큰 장점이었다. 출석과 과제 제출을 꼼꼼히 챙기면 환급을 받을 수 있고, 나도 직장을 다니면서 환급 조건을 맞출 수 있었다. 덕분에 금전적인 부담을 줄이면서도 끝까지 과정을 따라갈 동기부여가 됐다.

무엇보다 현업 데이터 엔지니어와 직접 이야기할 수 있는 기회가 있다는 것만으로도 참여할 이유는 충분하다고 느꼈다. 우수 수료생이 되면 멘토님에게 이력서, 자기소개서, 포트폴리오 첨삭도 받을 수 있다고 하니, 데이터 엔지니어로 취업이나 이직을 준비하는 사람에게도 추천하고 싶다.

## 마무리

이번 부트캠프를 통해 가장 크게 얻은 것은 방향성이었다. 이전에는 회사에서 필요한 기능을 만들면서도 내가 맞는 방향으로 가고 있는지 확신이 부족했다. 하지만 현직 데이터 엔지니어들의 설명을 듣고, 직접 질문하고, 스터디를 통해 파이프라인을 함께 구현하면서 내가 보완해야 할 부분이 더 선명해졌다.

특히 데이터 엔지니어링은 단순히 도구를 잘 쓰는 일이 아니라, 데이터가 안정적으로 흐르고 있는지 계속 확인하고 운영하는 일이라는 점을 배웠다. 앞으로는 회사에서 구축한 데이터 플랫폼도 데이터 품질, 신선도, 모니터링 관점에서 더 개선해보려고한다.

나처럼 백엔드 개발을 하다가 데이터 플랫폼이나 데이터 파이프라인 업무를 맡게 된 사람, 또는 데이터 엔지니어로 커리어를 확장하고 싶은 사람이라면 메타코드 데이터 엔지니어 부트캠프가 좋은 출발점이 될 수 있을 것 같다.
